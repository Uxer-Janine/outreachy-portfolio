Phase 1 – Technical Work (Docling OCR Pipeline)

View Repository

In this phase, I worked on building and analyzing a document processing pipeline using Docling and OCR engines to extract structured data from multilingual documents.

Pipeline Overview

The goal was to convert unstructured documents (PDFs in different languages) into structured Markdown format that can be used for downstream processing and analysis.

Tools & Technologies

  • Docling (document processing pipeline)
  • Tesseract OCR
  • Easyocr
  • Python
  • Git
  • Command Line Interface (CLI)

Workflow

  1. Prepared multilingual documents (Swahili and French)
  2. Processed documents using Docling with OCR support
  3. Converted outputs into Markdown format
  4. Organized outputs based on document type and OCR engine
  5. Validated outputs for structure and readability

Outputs

The pipeline generated structured Markdown files from input PDFs. Outputs were organized clearly to allow comparison across documents and OCR processing results.

Key Observations

  • OCR performance varied significantly depending on document structure and language
  • Clean, well-formatted documents produced more accurate outputs
  • Noise and formatting inconsistencies affected readability of extracted text
  • Structured output (Markdown) makes downstream processing more effective

Reflection

This phase strengthened my understanding of document processing pipelines, OCR limitations, and the importance of validating outputs before using them in downstream AI workflows. It also highlighted the need for combining technical execution with analytical evaluation.

Data Analysis & Visualizations

View Analysis Notebook

Beyond document processing, I conducted analysis on OCR outputs to better understand the quality, structure, and reliability of extracted text across different documents.

Analysis Approach

To move beyond surface-level observations, I analyzed OCR outputs using basic statistical and visual techniques. The goal was to evaluate not just how much text was extracted, but how usable and structured that text was.

Techniques Used

  • Word count comparison across documents
  • Line length distribution to assess structural patterns

These visualizations were used to better understand patterns, structure, and inconsistencies in OCR outputs across different documents.

Key Insights

  • Similar word counts do not necessarily indicate similar output quality
  • Document structure significantly affects OCR performance
  • Structured documents (e.g., phrasebooks) produce more predictable outputs than paragraph-based text
  • Noise and formatting inconsistencies can impact downstream usability

Reflection

This analysis reinforced the importance of validating OCR outputs before using them in downstream applications. It also highlighted how combining technical execution with analytical evaluation leads to more reliable and meaningful results.
